**核查**（Review）是你对摄像头上发生的事情进行分类整理的地方。它将活动分组为**核查项**——单个摄像头上一段时间内的片段，将同时活动的目标和音频打包在一起——并分为**警报**、**检测**和**画面变动**。从这里你可以拖动浏览活动、将项目标记为已核查、筛选、导出，以及跳转到[历史记录](/usage/history)中的完整录像。

本页介绍如何_使用_核查视图。有关警报和检测的_配置_方式（标签、区域、必需区域、保留），请参阅[核查配置](/configuration/review)文档。

:::info

只有当摄像头的**目标追踪和录制都已启用**时，才会为该摄像头创建核查项。请参阅[录制](/configuration/record)。

:::

## 警报、检测和画面变动

并非 Frigate 捕获的每段视频都具有相同的重要性。进入你物业的人可能比只是走在人行道上的人更重要。因此，Frigate 将**核查项**按重要性分为**警报**和**检测**，另有一个单独的**画面变动**类别用于显著的画面变动。

页面顶部的切换按钮在这三种严重程度之间切换。始终有一个被选中。

| 选项卡         | 指示颜色 | 显示内容                                                                                              |
| -------------- | -------- | ----------------------------------------------------------------------------------------------------- |
| **警报**       | 深红色   | 你最想看到的活动。默认情况下，所有 `person` 和 `car` 追踪目标都是警报。                               |
| **检测**       | 橙色     | Frigate 追踪的、未被提升为警报的其他所有内容。                                                        |
| **画面变动**   | 黄色     | 显著画面变动的时段，可筛选**未**产生追踪目标的时段。                                                  |

同样的颜色编码用于选中项的圆环和日历上的圆点。目标如何被归类为警报与检测——以及必需区域如何细化这一分类——在[警报和检测](/configuration/review#alerts-and-detections)中介绍。

**警报**和**检测**选项卡的标签旁会显示计数。当**显示已核查**关闭时（默认），这是仍待核查的项目数；打开时，计数反映所选时间范围内的所有项目。

## 将项目标记为已核查

核查项以缩略图卡片网格形式显示在垂直活动时间线旁。悬停卡片（桌面端）或向右滑动（移动端）会内联播放简短预览。

- **点击**卡片可在[历史记录](/usage/history)中打开其录像，并将该核查项标记为已核查。
- 每张卡片上的目标标记在未核查时为**灰色**，核查后变为**绿色**。
- **将这些项目标记为已核查**按钮可一次性将当前显示的所有项目标记为已核查。

已核查状态按用户跟踪，因此将一个项目标记为已核查不会对其他用户隐藏它。标记为已核查不会删除任何内容——录像和核查项本身会保留直到它们通过保留期过期。

## 选择和操作多个项目

要对多个项目进行操作，通过 **Ctrl/Cmd 点击**卡片（桌面端）或**长按**一个（移动端）开始选择。选中的卡片会获得与其严重程度匹配的彩色圆环。键盘快捷键可加快操作：`Ctrl+A` 全选，`R` 将选择标记为已核查，`Esc` 清除选择。

选择项目后，操作栏会显示以下选项：

- **导出**所选项目（单个项目直接导出；多个项目打开批量[导出](/usage/exports)对话框），
- **标记为已核查**或**标记为未核查**，以及
- **删除**（仅限管理员）。

## 筛选和日历

使用顶部的筛选控件缩小显示范围。可用筛选器取决于选项卡：警报和检测可按**摄像头**、**日期**、**标签**、**区域**以及项目是否已核查进行筛选；画面变动选项卡可按**摄像头**、**日期**和**仅变动画面**进行筛选。

**日历**筛选器可让你跳转到特定日期（在选择前显示**过去 24 小时**）。每天：

- 日期数字下方的**下划线**表示该天**有录像**。没有录像的日期会变暗。
- 日期数字下方的**彩色圆点**表示该天有**未核查活动**——**红点**表示未核查的警报，或当没有未核查警报时**橙点**表示未核查的检测。画面变动不用圆点表示。

未来日期被禁用，一周的开始和时区遵循你的配置。

## 核查画面变动

核查页面还可以显示未产生追踪目标的画面变动时段，并提供一种在特定区域内搜索过去录像中画面的方式。这些工具是对上述警报和检测工作流的补充——有关底层画面变动检测器的配置方式，请参阅[调整画面变动检测](/configuration/motion_detection)。

**画面变动**选项卡本身显示一个多摄像头网格，拖动到共享时间点，带有可拖动的时间线和播放速度选择器。当核查项或显著画面变动与当前时间重叠时，摄像头图块会获得彩色圆环，点击图块可在该时刻打开该摄像头的录像。每个摄像头的选项菜单（其图块角落的 kebab 菜单）是打开**画面变动预览**和**画面变动搜索**的地方，如下所述。

### 画面变动预览

画面变动预览面板显示未产生追踪目标的显著画面变动时段的预览片段。它有助于发现画面变动检测拾取到但目标检测未拾取的内容，这有助于验证调优或发现遗漏的目标。

在核查 > 画面变动页面上，点击摄像头上的 kebab 菜单并选择**画面变动预览**。每张卡片代表一个连续的仅画面变动活动范围，并播放该范围的录像预览。热力图叠加层会使无画面的区域变暗，以突出移动区域。

面板提供几个控制：

- **速度**——一次性加快或减慢所有预览片段。
- **变暗**——控制热力图叠加层对非画面区域的变暗程度。值越高，画面区域可见性越高。
- **筛选**——在摄像头快照上叠加 16×16 网格。选择一个或多个单元格，仅显示这些区域有画面的片段。这有助于过滤掉繁忙街道等区域的画面，同时保留车道上的画面。

点击预览片段可将录像播放器定位到该时间戳，以查看完整录像。

### 画面变动搜索

画面变动搜索可让你在摄像头上绘制的感兴趣区域内扫描录像中的变化。与画面变动预览（实时显示 Frigate 画面变动检测器标记的内容）不同，画面变动搜索会重新分析保存的录像，因此它可以找到被遗漏的变化（例如，在画面变动检测被 `lightning_threshold` 暂停时出现的目标，或通常被画面遮罩覆盖的区域中的变化）。

要开始搜索，在[历史记录](/usage/history)中打开操作菜单，或在核查 > 画面变动页面上点击摄像头上的 kebab 菜单并选择**画面变动搜索**。在对话框中：

1. 选择要扫描的摄像头和时间范围。在日期选择器中，有录像的日期会显示下划线。
2. 在摄像头画面上绘制多边形以定义感兴趣区域。
3. 如需要，调整搜索参数：

| 字段                | 说明                                                                                                                                                             |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **灵敏度阈值**      | 感兴趣区域内每像素亮度变化需达到的值才算作画面变动。行为类似于 Frigate 的画面变动检测 `threshold` 设置。                                                          |
| **最小变化区域**    | 单个移动区域的最小尺寸（占感兴趣区域的百分比），才算作显著。提高此值可忽略小幅度移动（树叶、远处的移动）；当目标仅占感兴趣区域的一小部分时降低此值。每个结果都会显示其得分百分比，因此你可以使用这些值来调优。 |
| **最大结果数**      | 返回匹配时间戳的最大数量。搜索在达到此数量后停止，因此较低的值完成更快，而较高的值会扫描更多范围。                                                                |
| **并行模式**        | 同时解码多个录像范围。加快大时间范围的搜索速度，但会消耗更多解码和 CPU 资源。                                                                                     |

画面变动搜索会自动采样每个录像的关键帧，因此无需调优帧率或采样设置。

运行后，Frigate 会扫描与时间范围重叠的录像片段，报告在多边形内检测到变化的时间戳以及变化占感兴趣区域的百分比。点击结果可将播放器定位到该时刻以查看发生了什么。

结果面板显示正在扫描的时间范围、带有当前正在分析的时间戳的实时进度条和运行中的结果计数。可折叠的**搜索指标**部分报告扫描和处理了多少片段、因感兴趣区域中无记录画面（使用存储的画面热力图）而跳过了多少片段、解码了多少帧以及总搜索时间。跳过所选感兴趣区域中无记录画面的片段是使长时间范围搜索变得可行的原因。

#### 常见用例

Frigate 的主要用例是录制和展示追踪目标，因此画面变动搜索对于目标检测未产生任何结果的情况最为有用——在浏览中没有目标可找，但你怀疑发生了某些事情。

- **定位未归属的变化。** 你知道在一段录像中出现了、消失了或移动了某些东西——包裹不见了、门被打开了——但没有检测指向它。搜索返回候选时间戳，而不是手动拖动时间线。
- **从未被检测到的目标。** Frigate 没有模型标签的某些东西、太小或太远而无法检测到的目标，或在未运行检测的区域中的移动。活动没有留下追踪目标，但确实改变了像素，因此搜索仍然可以找到它。
- **检测实际上被暂停时的活动。** 在目标检测被禁用、画面被 `skip_motion_threshold` 抑制或被画面遮罩覆盖的区域中发生的变化不会显示为核查项或追踪目标，但可以通过直接搜索录像来恢复。

#### 示例

这些示例展示了如何为两个常见目标选择感兴趣区域和**最小变化区域**。最小变化区域是单个移动区域的尺寸占你绘制的感兴趣区域的百分比，因此正确的值取决于你的目标——及其在采样之间的移动——覆盖了感兴趣区域的多少。

由于采样间隔为一秒或更长，移动目标通常在比较中同时出现在两个位置，因此即使是普通移动也经常得分几十个百分点，低阈值几乎会放行所有内容。最可靠的方法是**运行一次搜索，查看每个结果得分百分比，并将最小变化区域设置为略低于你关心的事件的值。** 默认值为 20%；以下建议仅供参考。

- **这个物品何时首次出现（或消失）？** 包裹被送达、车辆停放或垃圾桶被移动，你想要确切时刻。在物品占据的位置周围绘制**紧凑的感兴趣区域**并**提高最小变化区域**（从 40-60% 开始）。因为物品占据了紧凑感兴趣区域的大部分，其到达或移除是一个大变化，而附近较小的移动（阴影、路过的行人）保持在阈值以下。**最早的结果**是它出现的时刻；如果你只关心那一刻，较低的最大结果数完成更快。如果没有命中，感兴趣区域可能比物品大——降低阈值或收紧感兴趣区域。
- **什么进入了花园？** 某物夜间践踏了花坛且从未追踪到目标。绘制一个覆盖整个花坛的**较松散的感兴趣区域**，使用**比上述情况更低的最小变化区域**——从 20% 默认值附近开始，仅在小型或远处目标被遗漏时降低（趋向 5-10%），因为它只占大区域的一小部分。预期会有更多结果需要浏览——逐步查看时间戳并跳转到每个结果以查看触发原因。如果风吹植物产生噪声，请提高最小变化区域或灵敏度阈值。

#### 预期性能

画面变动搜索按需分析保存的录像而非读取预建索引，因此长时间范围的搜索比浏览画面变动预览需要更长时间。成本主要随需要检查的录像量而变化：使用存储的画面热力图跳过感兴趣区域中无记录画面的片段（在状态面板中显示为"跳过的片段"），因此安静范围完成快，繁忙范围完成慢。

提高搜索速度的方法：

- 绘制紧凑的感兴趣区域。由于**最小变化区域**以你绘制区域的百分比衡量，在预期变化位置周围绘制紧凑的感兴趣区域会使目标占据更大比例的区域，因此更容易超过阈值。松散的感兴趣区域使同一目标成为区域的一小部分，可能低于阈值而被遗漏——迫使你降低最小变化区域，这会放入更多噪声。
- 将时间范围缩小到你关心的窗口，以减少需要检查的录像。
- 当你只需要前几个结果时降低**最大结果数**。因为搜索在达到该数量后停止，较小的值可以让繁忙范围提前完成，而不是扫描整个窗口。
- 使用并行模式在多核系统上缩短实际时间，但运行时会消耗更多解码和 CPU 资源。

## AI 核查摘要

当配置了[生成式 AI 核查](/configuration/genai/genai_review)后，Frigate 可以为核查项生成标题、描述和威胁分类，并自动在核查和历史记录中显示。点击摘要标记可打开 **AI 分析**对话框，显示生成的详情和任何标记的关注点。

在核查中，AI 分类为**可疑**（Level 1）或**严重**（Level 2）的未核查项上会出现额外的图标，以便在你打开之前最值得关注的活动就凸显出来。项目被核查后图标会消失。
